# Listagem de usuários

## Objetivo

A aplicação Studio deve apresentar uma página de gestão de usuários cadastrados para ser possível ter uma visão geral de todos os usuários cadastrados no sistema.

## Funcionalidades

### Tabela de usuários

- [x] Ter as colunas de nome, nível, xp semanal, estela desbloqueadas, conquistas desbloqueadas, desafios completados, status do espaço, insígnias e data de criação.
- [] Ser listado no máximo 20 usuários em forma de tabela
- [] Ter a opção de filtrar os usuários por nome.
- [] A paginação deve ser de tal forma que o usuário possa navegar entre as páginas por botões de próxima e anterior, quanto de número.
- [] Deve ser possível o usuário escolher quantos usuários devem ser exibidos por página, sendo as opções de 10, 20, 50. O padrão deve ser 20.

## Fluxo de navegação

- Visualizar a sidebar com os links de navegação.
- Visualizar a o link de navegação para a página de usuários.

## Fluxo de filtragem por nome

- Começar a digitar o nome do usuário em um campo de busca.
- Os usários correspondentes ao nome digitado são exibidos.

## Fluxo de paginação

- Usuário vizualisa os botões de paginação abaixo da tabela de usuários.
- Usuário clica em um dos botões de paginação numérico um dos botão de próximo e anterior.
- Os usários correspondentes a página são exibidos.
